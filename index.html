Funciones de Git

El uso de Git

Se conoce a git como un sistema de control de versiones capaz de ejecutar comandos, permitiendo realizar operaciones de alto nivel como el acceso completo a componentes internos.

¿Qué son los comandos?

Los comandos son órdenes que se utilizan para realizar acciones en un repositorio de Git

Listado de comandos principales para el funcionamiento:

Git clone:

Sirve para descargar el código fuente existente desde un repositorio remoto.

Git commit:

Sirve para guardar un archivo modificado de manera local, no es neceario que se guarde en remoto.

Git branch:

Sirve para crear ramoas y así poder crear vesriones que no afecten directamente al codigo fuente Main/Master.

Git checkout:

Sirve para cambiar entre ramas y poder empezar a trabajar en ella, al igual, sirve para revisar archivosy commits.

Git status:

sirve para dar toda la información necesaria sobre la rama actual

Git add:

Cuando se crea, edita, elimina o modifica, estos cambios ocurren en los archivos locales y no se incluirán siguiente commit a menos que se cambie la configuración.

Sirve para incluir los cambios realizados en local al siguiente commit

Git push:

Se encarga de enviar los cambios realizados en el código local al servidor remoto. Envía los commits al repositorio remoto

Git pull:

Sirve para recibir actulzaciones del repositorio remoto.
Este comando combina git fetch y git merge, lo que significa que recogeremos actualizaciones del repositorio remoto e inmediatamente aplicamos estos últimos cambios
